# Codex 작업 지침

## 프로젝트 맥락 참고

- 이 저장소 전체의 현재 구조와 요약은 `ai-index/project_overview.json`을 우선 참고한다.
- 파일별 최신 요약, 키워드, 태그, 관련 주제는 `ai-index/summary.json`을 함께 확인한다.
- 사용자가 세컨드 브레인 전체 구조, 프로젝트 성격, 현재 저장된 내용, 파일별 요약을 물으면 `ai-index/project_overview.json`을 프로젝트 소스로 사용한다.
- 세컨드 브레인의 큰 구조나 핵심 내용이 바뀌면 `ai-index/project_overview.json`도 함께 갱신한다.

## 수학 노트 작성

- 수학 노트, 특히 `학업/과목` 아래의 수학 과목 정리는 LaTeX 수식 문법으로 작성한다.
- 인라인 수식은 `$...$` 형식을 사용한다.
- 독립 수식은 `$$...$$` 형식을 사용한다.
- 계산 과정, 정의, 정리, 공식은 가능한 한 한국어 설명과 LaTeX 수식을 함께 적는다.
- Obsidian에서 바로 렌더링 가능한 Markdown/LaTeX 문법을 유지한다.

## 문서 정리 원칙

- `홈.md`에는 최상위 허브 이름만 적고 위키링크를 만들지 않는다.
- 최상위 허브는 `개인`, `일정`, `학업`, `자산과 일`, `AI`, `개발`, `취미`, `자료와 참고`를 기본으로 한다.
- Obsidian 문서의 위키링크는 원칙적으로 `상위 문서:` 한 줄에만 사용한다.
- 본문에서는 문서명, 과목명, 인물명, 도구명, 채널명, 종목명, 프로젝트명을 위키링크로 만들지 않는다.
- 보조 연결용 섹션을 만들지 않는다.
- 과목 문서에는 수업 정보, 시험 일정 같은 상위 정보만 둔다.
- 쪽지시험 복기, 오답 정리, 개별 학습 기록은 과목 문서에 직접 길게 쓰지 말고 별도 문서로 만든다. 별도 문서의 `상위 문서:`에서 과목 문서로 연결한다.
- `일정/중요 날짜.md`는 월별 주요 일정을 한곳에 모아 관리한다.
- 약속, 행사, 개인 일정은 내용이 짧으면 별도 문서로 쪼개지 말고 `일정/중요 날짜.md`에 직접 기록한다.
- 일정 세부 정보가 길어지거나 관련 자료가 많아질 때만 별도 문서를 만들고, 별도 문서의 `상위 문서:`에서 일정 문서로 연결한다.
- 개인 일상 기록은 날짜 단위 문서로 관리한다.
- `개인/일상 기록.md`는 날짜별 일상 기록 문서를 모아두는 인덱스 문서로 사용한다.
